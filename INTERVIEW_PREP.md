# Interview Prep: Eyewear Detection Pipeline

## 1) Что за проект
`Eyewear Detection Pipeline` решает задачу определения наличия очков на лице в изображении/видео.
Пайплайн двухшаговый:
1. Детекция лица.
2. Классификация кропа лица (`glasses/no_glasses`).

Почему так:
- легче отлаживать;
- проще объяснять ошибки;
- удобнее для постепенного улучшения.

## 2) Ключевые инженерные решения
- baseline модель для быстрой валидации идеи и регрессионного контроля;
- torch-модель (MobileNetV3) как основной ML baseline;
- единые манифесты и сплиты для воспроизводимости;
- метрики качества + confusion matrix;
- API и Docker для демонстрации в формате продукта.

## 3) Что говорить про метрики
- Основная метрика: `F1` по классу `glasses`.
- Бизнес-аспект:
  - высокий `recall` уменьшает пропуски людей в очках;
  - высокий `precision` уменьшает ложные срабатывания.
- Threshold влияет на баланс `precision/recall`.

## 4) Где модель ошибается
- плохое освещение;
- профильный ракурс;
- закрытые/частично закрытые глаза;
- маленькие лица в кадре.

## 5) Как улучшать
1. Улучшить face detector (RetinaFace/MTCNN).
2. Добавить real-world датасет и hard examples.
3. Тонко настроить аугментации и порог классификации.
4. Рассмотреть третий класс `sunglasses`.
5. Оптимизировать инференс через ONNX Runtime.
